# 共通仕様

各機能の仕様書はこのドキュメントを前提とする。個別の仕様書では、ここに記載の内容は参照のみとし重複記載しない。

## ページネーション

一覧取得系のエンドポイントで共通のクエリパラメータ・レスポンス形式。

**クエリパラメータ**

| 名前 | 型 | 必須 | 説明 |
|---|---|---|---|
| page | int | 任意 | ページ番号（デフォルト: 0） |
| size | int | 任意 | 1ページ件数（デフォルト: 20、最大: 100） |

**レスポンス形式**

```json
{
  "content": [ /* 各要素 */ ],
  "totalElements": 0,
  "totalPages": 0
}
```

## エラーレスポンス形式

```json
{
  "status": 400,
  "message": "Validation failed",
  "errors": [
    { "field": "title", "message": "must not be blank" }
  ]
}
```

`errors` はバリデーションエラー（400）の場合のみ含む。

## 共通処理

### 参照先IDの存在チェック

リクエストに他エンティティのIDを含む場合（例: `authorIds`, `categoryIds`）、登録・更新時に以下を行う。

1. 指定された全IDが対象テーブルに存在するか確認する
2. 存在しないIDが1件でもあれば `400 Bad Request` を返す（`errors` に該当フィールドと不正なIDを含める）

### 一意制約違反

一意制約のある項目（例: ISBN、メールアドレス）が重複した場合は `409 Conflict` を返す。

### 参照整合性による削除制限

他のレコードから参照されているエンティティを削除しようとした場合は `409 Conflict` を返す。（例: 貸出中の書籍、貸出履歴のある会員）

## 共通ステータスコード

| ステータス | 意味 |
|---|---|
| 200 OK | 取得・更新成功 |
| 201 Created | 登録成功 |
| 204 No Content | 削除成功 |
| 400 Bad Request | バリデーションエラー、または参照先ID不正 |
| 404 Not Found | 指定したリソースが存在しない |
| 409 Conflict | 一意制約違反、または参照整合性による削除制限 |
