# 会員機能 仕様書

## 概要

図書館利用者（会員）の登録・参照・更新・削除を行う機能。貸出とは一対多で関連付けられる。

共通のページネーション・エラー形式・ステータスコードは [common.md](./common.md) を参照。

## 一覧取得

```
GET /members
```

**クエリパラメータ**

| 名前 | 型 | 必須 | 説明 |
|---|---|---|---|
| page, size | - | 任意 | [共通仕様](./common.md#ページネーション)参照 |

**処理ステップ**

1. クエリパラメータを受け取る
2. `page` / `size` 未指定時は[共通仕様](./common.md#ページネーション)のデフォルト値を適用
3. 会員を登録日時の降順で取得しページング処理を行う
4. `200 OK` で返却する

## 詳細取得

```
GET /members/{id}
```

**処理ステップ**

1. `id` に一致する会員を取得する
2. 存在しない場合は `404 Not Found` を返す
3. 存在する場合、`200 OK` で返却する

## 登録

```
POST /members
```

**リクエストボディ**

```json
{
  "name": "山田太郎",
  "email": "yamada@example.com",
  "phone": "090-1234-5678"
}
```

**処理ステップ**

1. リクエストボディを受け取る
2. 必須項目を検証する。不正な場合は `400 Bad Request`（[共通エラー形式](./common.md#エラーレスポンス形式)）
   - `name`: 必須、1〜255文字
   - `email`: 必須、メール形式
   - `phone`: 任意
3. `email` の一意性を確認する。既に登録済みの場合は `409 Conflict`（[共通処理](./common.md#一意制約違反)）
4. 会員レコードを `registered_at` に現在日時を設定して `INSERT` する
5. `Location: /members/{id}` ヘッダーを付与して `201 Created` を返す

## 更新

```
PUT /members/{id}
```

- リクエストボディは登録時と同じ形式（`registered_at` は変更不可）

**処理ステップ**

1. `id` に一致する会員が存在するか確認する。存在しない場合は `404 Not Found`
2. リクエストボディを登録時と同じルールで検証する。不正な場合は `400 Bad Request`
3. `email` を変更する場合、変更後の値が自分以外の会員と重複していないか確認する。重複していれば `409 Conflict`
4. 会員レコードを `UPDATE` する（`registered_at` は更新しない）
5. `200 OK` を返す

## 削除

```
DELETE /members/{id}
```

**処理ステップ**

1. `id` に一致する会員が存在するか確認する。存在しない場合は `404 Not Found`
2. 当該会員に紐づく `loans` レコードが1件でも存在しないか確認する（貸出履歴を含む）。存在する場合は `409 Conflict`（[共通処理](./common.md#参照整合性による削除制限)）
3. 会員レコードを削除する
4. `204 No Content` を返す
